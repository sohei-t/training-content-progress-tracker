# エージェント用 自動実行指示書

## アプリ開発が完了したら

### 1. リリース版の作成（エージェントが実行）
```bash
# 開発が完了したらプロジェクトルートで実行
./release.sh
```
これにより、`~/Desktop/my-apps/` に独立版が作成されます。

### 2. GitHubへの公開（エージェントが実行）
```bash
# リリース版作成後に実行
./publish_to_portfolio.sh
```
これにより、GitHubポートフォリオに自動公開されます。

## 重要な判断ポイント

エージェントは以下を確認してから実行してください：

1. **テストが全て通っているか**
   - `npm test` または `pytest` が成功

2. **基本機能が動作するか**
   - アプリが起動する
   - エラーが出ない

3. **ドキュメントが生成されているか**
   - README.md が存在
   - 可能ならHTMLスライドと音声も生成

## 自動実行の流れ

```yaml
開発完了の判定:
  - ユーザーから「完成」「リリースして」等の指示
  - または全要件が実装完了

自動実行:
  1. テスト実行 → PASS確認
  2. ./release.sh 実行
  3. ./publish_to_portfolio.sh 実行
  4. 完了報告とURL提示
```

## エージェントへの指示例

ユーザー: 「電卓アプリを作って、GitHubに公開して」

エージェント:
1. アプリ開発を実施
2. テスト確認
3. 自動的に `./release.sh` を実行
4. 自動的に `./publish_to_portfolio.sh` を実行
5. 「公開完了しました！URL: https://github.com/...」と報告
